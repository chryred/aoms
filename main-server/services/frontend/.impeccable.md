# .impeccable.md — Synapse-V / Synapse-V Design Context

## Design Context

### Users
운영팀 엔지니어 및 시스템 관리자. 24시간 모니터링 환경에서 사용하며, 장애 감지·대응·보고가 주요 목적이다.
스트레스가 높은 상황(알림 발생, 이상 탐지)에서도 빠르게 상황을 파악할 수 있어야 한다.
신뢰감과 명확성이 미적 즐거움보다 우선한다.

### Brand Personality
- **제품명**: Synapse-V (기존 "Synapse-V" 레이블 대체)
- **서체 전략**: Synapse-V 브랜드 워드마크에만 별도 영어 서체 사용 (예: Inter Display, Syne 등 기하학적 sans-serif)
- **3단어**: Precise · Trustworthy · Composed
- **감성 목표**: 장애 상황에서도 차분함과 통제감을 준다. "이 시스템이 상황을 파악하고 있다"는 느낌.

### Aesthetic Direction
- **유지**: 뉴모피즘 (soft 3D shadow, #E8EBF0 베이스) — 방향은 유지하되 완성도와 세련도 향상
- **제거**: 과도한 gradient, glow, AI-스러운 보라/파랑 빛 번짐 효과
- **추가 금지**: 과도한 애니메이션, 비즈니스 목적에 맞지 않는 playful 요소
- **참고 레퍼런스**: Linear (정밀함, 여백), Bloomberg Terminal (정보 밀도와 신뢰감), AWS Console (엔터프라이즈 안정감)
- **안티 레퍼런스**: ChatGPT UI, Vercel AI 제품군 (gradient glow, purple haze)
- **테마**: 라이트 모드 전용

### Design Principles

1. **데이터가 주인공이다** — UI 크롬(테두리, 장식, 그림자)은 데이터 판독을 방해해서는 안 된다. 장식적 요소는 최소화하고 정보 위계를 최우선한다.

2. **신뢰는 일관성에서 온다** — 같은 역할을 하는 요소는 항상 같은 스타일을 써야 한다. 색상, 간격, 폰트 굵기의 임의적 변화는 금지.

3. **긴급도는 색상으로만 전달한다** — critical(빨강), warning(주황), normal(초록)의 시맨틱 색상은 절대 장식 목적으로 사용하지 않는다.

4. **여백은 설계된 것이다** — 공간이 충분해야 빠른 스캔이 가능하다. 정보 밀도를 높이기 위해 여백을 줄이는 것은 오히려 가독성을 해친다.

5. **브랜드는 절제 속에 드러난다** — Synapse-V 워드마크에만 전용 서체를 사용하고, 나머지 UI는 Pretendard 단일 폰트를 유지한다. 브랜드 색상(indigo #6366F1)은 핵심 인터랙션 요소에만 집중적으로 사용한다.

## Tech Stack

- **Framework**: React 18 + TypeScript, Vite 6
- **Styling**: TailwindCSS 4.x (utility-first, no CSS-in-JS)
- **Design Pattern**: Neumorphism
- **Primary Font**: Pretendard (Korean-optimized, woff2)
- **Brand Font**: TBD — 영어 기하학적 sans-serif (Synapse-V 워드마크 전용)
- **Icons**: lucide-react
- **Charts**: recharts
- **State**: Zustand + @tanstack/react-query
