---

## SLIDE 1 — 표지

**Synapse-V**
백화점 통합 IT 모니터링 시스템

> "안녕하십니까. 백화점CX팀에서 개발한 통합 모니터링 시스템, Synapse-V를 소개드리겠습니다."

---

## SLIDE 2 — 왜 만들었나

| 현실 | 결과 |
|---|---|
| 담당자가 각자 시스템을 수동 확인 | 장애 인지까지 평균 15~30분 |
| 로그 수동 확인 → 원인 파악 지연 | 고객 불편·매출 손실 직결 |
| 내부망 환경 | 클라우드 솔루션 도입 불가 |
| 퇴근 후·주말 수동 대응 | 야간 장애 공백, 운영 피로 누적 |

> "장애가 나면 서버에 직접 붙어 로그를 뒤지고, 원인을 분석하는 동안 처리시간은 지연되고 있었습니다."

---

## SLIDE 3 — 무엇을 만들었나

> 잠들어 있어도 장애를 먼저 발견하고, 원인과 해결책을 들고 찾아오는 시스템

**핵심 특징**
- 내부망 100% 지원 — 내부망만으로 완전 동작
- 데이터 보안 — 로그가 외부로 나가지 않는 내부 DevX AI 분석
- 설치 간편 — 버튼 클릭 한 번으로 에이전트 자동 배포

## SLIDE 4 — 전체 기능 한눈에

| # | 기능 | 핵심 내용 |
|---|---|---|
| ① | **AI 로그 분석** | 이상 원인 자동 분류, 수십 분 → 수 초 |

## SLIDE 5 — 주요 기능 ① AI 로그 분석

**"로그를 읽어주는 AI"**

| 구분 | 기존 | Synapse-V |
|---|---|---|
| 원인 파악 | 담당자 수백 줄 직접 확인 (20~30분) | AI 자동 분석 (수 초) |
| 알림 방식 | 없음 (수동 발견) | 담당자 Teams 자동 @mention |

```
원인: DB Connection Pool 고갈 — 세션 수 임계 초과
심각도: CRITICAL
담당자: 홍길동 팀장 → Teams 자동 알림
```

**핵심 효과**: 장애 인지·원인 파악을 동시에, 사람 개입 없이

---

## SLIDE 6 — 주요 기능 ② 유사 장애 검색

**"같은 장애, 두 번 고생하지 않는다"**

- 과거 장애 이력을 AI가 자동 비교 → 유사 해결책 즉시 제안
- 해결 후 [등록] 버튼 한 번 → 조직 지식 자산으로 자동 저장

```
현재 장애 발생
  → 유사도 87% 매칭
  → "2025-11-03 동일 증상 — 해결책: WAS 재기동 후 Connection Pool 조정"
```

**핵심 효과**: 신규 인력도 즉시 베테랑 수준 대응 / 개인 경험 → 조직 자산

---

## SLIDE 7 — 주요 기능 ③ Teams 실시간 알림

**"알림이 끝이 아닌 시작"**

```
🔴 [CRITICAL] 고객경험시스템 / WAS1
원인: DB Connection Pool 고갈
유사 장애: 87% 일치 (2025-11-03)
해결책: WAS 재기동 → Pool 설정 조정

[해결책 등록] → 조직 지식 베이스 자동 저장
```

- 시스템별 담당 채널로 분리 발송
- 복구 완료 알림도 자동 발송

**핵심 효과**: 장애 대응 커뮤니케이션 비용 약 60% 절감

---

## SLIDE 8 — 주요 기능 ④ 장애 예방 분석

**"다음 장애를 미리 본다"**

| 주기 | 분석 내용 |
|---|---|
| 1시간 | 메트릭 이상 패턴 + AI 단기 예측 |
| 1일 | 장애 빈도 + 피크 시간대 분석 |
| 1주 | 반복 이상 항목 식별 |
| 1월 | 중장기 용량 예측 + 월간 리포트 |

```
⚠️ 고객경험시스템 CPU — 3일 연속 92% 초과
→ 주말 전 사전 증설 권고
```

**핵심 효과**: 사후 대응 → 사전 예방으로 전환

```
┌─────────────────────────────────────────────────────────────────────┐
│                          폐쇄망 내부                                  │
│                                                                      │
│  [대상 서버]                [Server A — 메인]                         │
│  ┌─────────────┐           ┌──────────────────────────────────────┐ │
│  │ WAS 서버    │ ─에이전트─▶│ Prometheus   메트릭 수집·저장         │ │
│  │ DB 서버     │ ─에이전트─▶│ Loki         로그 수집·저장           │ │
│  │ 미들웨어    │ ─에이전트─▶│ Grafana      시각화 대시보드          │ │

│                             └──────────────────────────────────────┘ │
│                                          │                           │
│                                          ▼                           │
│                             [Server B — AI·벡터]                     │
│                             ┌──────────────────────────────────────┐ │
│                             │ Ollama   내부 LLM (AI 분석·챗봇)      │ │
│                             │ Qdrant   벡터 DB (유사 장애 검색)     │ │
│                             └──────────────────────────────────────┘ │
│                                          │                           │
└──────────────────────────────────────────┼───────────────────────────┘

                              담당자 채널 실시간 알림
```

**서버 사양 요건**

| 서버 | 역할 | 최소 사양 |
|---|---|---|
| Main-Server | 수집·저장·분석·관리 UI | 4코어 / 8GB RAM |
| Sub-Server | AI 추론·벡터 검색 | 4코어 / 8GB RAM |
| 대상 서버 | 에이전트만 설치 (4MB) | 추가 사양 불필요 |
