# 📘 LLM 기반 레거시 시스템 모니터링  
## 수집기(Agent) 설치·제어 요구사항 정의서

---

## 1. 문서 목적

본 문서는 **레거시 시스템(JEUS 기반 모놀리식 환경)** 에서  
로그·메트릭·JVM 지표 수집을 위한 **Agent(Alloy, Node Exporter, JMX Exporter)** 를  
**웹 기반 화면(Web UI)** 에서 **비‑root 권한으로 설치·제어·검증**하기 위한  
기능적/비기능적 요구사항을 정의한다.

---

## 2. 적용 범위 및 전제 조건

### 2.1 적용 대상 시스템

| 항목 | 내용 |
|------|------|
| WAS | JEUS |
| JDK | 1.7 |
| 아키텍처 | 모놀리식 |
| 배포 구조 | 단일 서버 내 다중 컨테이너 / WAS 이중화 |
| 운영 제약 | 서버별 접근 권한 상이, root 상시 사용 불가 |

---

### 2.2 기본 전제

- 애플리케이션 **코드 수정 없음**
- 서버 **Inbound 포트 개방 최소화**
- 수집기는 **서버 Outbound 통신만 사용**
- 계정 정보는 **일회성 사용 (저장 금지)**

---

## 3. 전체 아키텍처 개요

```
[Web UI]
 ├─ 시스템/서버/컨테이너 등록
 ├─ 수집기 설치/제어 버튼
 └─ 상태/검증 결과 조회
        ↓
[Control / Installer Backend]
 ├─ Job 생성
 ├─ 설정 템플릿 관리
 └─ 결과 수집
        ↓ (HTTPS / SSH / Agent Pull)
[Target Server]
 ├─ Alloy (컨테이너별)
 ├─ Node Exporter (서버별 1개)
 ├─ JMX Exporter (서비스별)
 └─ Health Check
        ↓
[Monitoring Stack]
 ├─ Loki
 ├─ Prometheus
 └─ LLM Analysis Layer
```

## 5. 수집기별 요구사항

---

## 5.1 Alloy (Log / 일부 Metrics 수집기)

### 5.1.1 설치 및 분리 정책

| 항목 | 요구사항 |
|---|---|
| 설치 단위 | ✅ 컨테이너 / 서비스 단위 |
| 서버 내 개수 | ✅ 다수 허용 |
| 실행 계정 | ✅ 컨테이너별 계정 |
| 권한 범위 | 해당 컨테이너 로그 디렉터리만 |

---

### 5.1.2 컨테이너 다중 구성 시 요구사항

- 서버 내 **컨테이너 수만큼 Alloy 설치 가능**
- 각 Alloy는 **서로 다른 계정으로 실행**
- 로그 접근 권한 충돌 방지
- 동일 서버임을 식별할 수 있는 **공통 server label 부여**

---

## 5.2 Node Exporter (시스템 자원 수집기)

### 5.2.1 설치 정책

| 항목 | 요구사항 |
|---|---|
| 설치 개수 | ✅ 서버당 1개 |
| 실행 계정 | ✅ 일반 계정 |
| 수집 범위 | 서버 전체 CPU / Memory / Disk / Network |

---

### 5.2.2 분리 및 연관관계 정책

> Node Exporter는 **시스템 자원을 물리적으로 분리 수집할 수 없음**

#### ✅ 요구사항 정의

- Node Exporter는 **서버 단위 지표만 수집**
- Alloy / JMX Exporter와 **논리적 연관관계 설정 필요**
- 동일한 `server_id` / `env` Label 사용

#### ✅ 활용 목적

- 컨테이너/서비스 장애 발생 시  
  → **서버 자원 상태와의 상관 분석**
- LLM 분석 시 **배경 자원 컨텍스트 제공**

---

## 5.3 JMX Exporter (JVM 지표 수집기)

### 5.3.1 제공 방식

| 항목 | 요구사항 |
|---|---|
| 제공 형태 | ✅ Standalone JMX Exporter |
| Java Agent 사용 | ❌ 사용하지 않음 |
| JEUS 재기동 | ❌ 불필요 |
| root 권한 | ❌ 불필요 |

---

### 5.3.2 서비스별 분리 요구사항

| 항목 | 요구사항 |
|---|---|
| 실행 단위 | ✅ JEUS 인스턴스 / 서비스별 |
| JMX 포트 | 서비스별 개별 설정 |
| Exporter 포트 | 서비스별 분리 |
| Label | service / app 단위 구분 |

---

## 6. 시스템 연계 및 관리 요구사항

### 6.1 시스템 등록 연계

- 모든 수집기는 **논리적 “시스템” 단위로 연계 등록**
- 예시:
  - 시스템 → 서버 → 컨테이너 → 수집기

---

### 6.2 화면 기준 관리 구조

| 레벨 | 관리 대상 |
|---|---|
| 시스템 | 업무/서비스 단위 |
| 서버 | 물리/가상 서버 |
| 컨테이너 | WAS / 배치 |
| 수집기 | Alloy / Node / JMX |

---

## 7. 보안 요구사항

| 항목 | 요구사항 |
|---|---|
| root 상시 사용 | ❌ 금지 |
| 계정 정보 저장 | ❌ 금지 |

| 접근 범위 | 최소 권한 원칙 |
| 로그 마스킹 | 필수 적용 |

## 8. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| 성능 영향 | CPU < 2%, Memory < 300MB |
| 확장성 | 서버/컨테이너 증가 대응 |
| 장애 격리 | 수집기 장애 ≠ 서비스 장애 |
| 롤백 | 즉시 제거 가능 |

---

## 9. 최종 정리 (핵심 원칙)

- ✅ **모든 수집기는 Web UI에서 제어**
- ✅ **root 권한 없이 설치/운영**
- ✅ **Node Exporter는 서버 단위, 논리적 연관**
- ✅ **Alloy는 컨테이너/계정 단위 분리**
- ✅ **JMX Exporter는 Standalone 방식**
- ✅ **설치 후 즉시 정상 동작 검증 가능**
