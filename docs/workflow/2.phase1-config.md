## Phase 1: 인프라 기반 구성

> **목표**: 모니터링 서버에 Docker 기반 PLG 스택을 구축하고, 파일럿 시스템(이중화 포함)에서 메트릭/로그 수집을 확인한다.

### 태스크 의존성 그래프

```
T1.1 (모니터링 서버 환경 준비)
  └─► T1.2 (Docker 이미지 Load)
        └─► T1.3 (프로젝트 구조 생성)
              └─► T1.4 (Prometheus 설정)
                    ├─► T1.5 (Alertmanager 설정)
                    ├─► T1.6 (Loki 설정)
                    └─► T1.7 (Grafana 설정)
                          └─► T1.8 (docker-compose.yml 작성)
                                └─► T1.9 (PostgreSQL 초기화 및 검증)
                                      └─► T1.10 (전체 스택 기동 & 헬스체크)
                                            └─► T1.11 (파일럿 에이전트 설치)
                                                  └─► T1.12 (수집 확인 & 대시보드)
                                                        └─► [CP1]
```

### T1.1 — 모니터링 서버 환경 준비 (RedHat 8.9)

```bash
# ---- 모니터링 서버 (RedHat 8.9)에서 실행 ----

# Docker 설치 (폐쇄망: RPM 패키지 사전 전송 필요)
# Mac에서 RPM 다운로드 후 전송
#   curl -LO https://download.docker.com/linux/rhel/8/x86_64/stable/Packages/docker-ce-26.1.4-1.el8.x86_64.rpm
#   (containerd, docker-ce-cli 등 의존 패키지도 함께 다운로드)
# 전송 후 설치:
sudo rpm -ivh /opt/aoms-offline/rpms/containerd.io-*.rpm
sudo rpm -ivh /opt/aoms-offline/rpms/docker-ce-cli-*.rpm
sudo rpm -ivh /opt/aoms-offline/rpms/docker-ce-*.rpm

sudo systemctl enable --now docker

# Docker Compose Plugin 설치
sudo mkdir -p /usr/local/lib/docker/cli-plugins
sudo cp /opt/aoms-offline/rpms/docker-compose-linux-x86_64 \
        /usr/local/lib/docker/cli-plugins/docker-compose
sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

# 현재 사용자를 docker 그룹에 추가
sudo usermod -aG docker $USER
newgrp docker

# 방화벽 설정 (내부 네트워크 IP 대역 제한 권장)
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="3000" protocol="tcp" accept'   # Grafana
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="9090" protocol="tcp" accept'   # Prometheus
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="9093" protocol="tcp" accept'   # Alertmanager
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="3100" protocol="tcp" accept'   # Loki
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="8080" protocol="tcp" accept'   # Admin API
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="10.0.0.0/8" port port="8081" protocol="tcp" accept'   # Feedback Server
sudo firewall-cmd --reload
firewall-cmd --list-rich-rules   # 적용 확인

# SELinux 상태 확인 (Enforcing이어야 함)
getenforce
# → Enforcing  ← Docker 볼륨 마운트 시 :z 옵션 필수

# 시간대 설정 확인
sudo timedatectl set-timezone Asia/Seoul
timedatectl status

# 디스크 여유 공간 확인
df -h   # / 파티션 50GB 이상 확인

# Python 가상환경 생성 (스크립트용)
python3 -m venv /opt/monitoring/venv
/opt/monitoring/venv/bin/pip install \
  --no-index \
  --find-links=/opt/aoms-offline/pip-packages/ \
  requests python-dateutil bcrypt flask
/opt/monitoring/venv/bin/pip list   # 설치 확인
/var/log/llm-analyzer
# 스크립트 및 데이터 디렉토리 생성
mkdir -p /opt/monitoring/scripts
mkdir -p /opt/monitoring/data
mkdir -p /var/log/llm-analyzer -> /app/aoms/logs/llm-analyzer

# Trivy 오프라인 설치 (보안 스캔용)
sudo rpm -ivh /opt/aoms-offline/agents/linux/trivy_0.69.3_Linux-64bit.rpm
mkdir -p ~/.cache/trivy/
cp -r /opt/aoms-offline/trivy-db/* ~/.cache/trivy/

# 반입된 Docker 이미지 보안 스캔 (오프라인 모드)
# trivy image --skip-update prom/prometheus:v3.10.0
# trivy image --skip-update grafana/grafana:12.4.0
# → Critical 0건 확인 후 진행

# Grafana HTTPS 자체 서명 인증서 생성
mkdir -p /opt/aoms/ssl
openssl req -x509 -newkey rsa:4096 \
  -keyout /opt/aoms/ssl/grafana.key \
  -out    /opt/aoms/ssl/grafana.crt \
  -days 3650 -nodes \
  -subj "/C=KR/ST=Seoul/L=Seoul/O=Company/CN=$(hostname -I | awk '{print $1}')" \
  -addext "subjectAltName=IP:$(hostname -I | awk '{print $1}')"
chown -R 472:472 /opt/aoms/ssl
chmod 600 /opt/aoms/ssl/grafana.key
```

`**config/prometheus/alert_rules.yml**`:

```yaml
groups:
  - name: resource_alerts
    rules:
      - alert: HighCpuUsage
        expr: |
          (1 - avg by(instance, system_name, instance_role, host) (
            rate(node_cpu_seconds_total{mode="idle"}[5m])
          )) * 100 > 80
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "CPU 사용률 80% 초과"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): CPU {{ $value | printf \"%.1f\" }}%"

      - alert: CriticalCpuUsage
        expr: |
          (1 - avg by(instance, system_name, instance_role, host) (
            rate(node_cpu_seconds_total{mode="idle"}[5m])
          )) * 100 > 90
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "CPU 사용률 90% 초과 (위험)"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): CPU {{ $value | printf \"%.1f\" }}%"

      - alert: HighMemoryUsage
        expr: |
          (1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100
            > 80
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "메모리 사용률 80% 초과"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): Memory {{ $value | printf \"%.1f\" }}%"

      - alert: HighDiskUsage
        expr: |
          (1 - node_filesystem_avail_bytes{fstype!~"tmpfs|overlay"}
               / node_filesystem_size_bytes) * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "디스크 사용률 80% 초과"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): {{ $labels.mountpoint }} {{ $value | printf \"%.1f\" }}%"

      - alert: ServiceDown
        expr: up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "에이전트 응답 없음"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): Exporter 다운"

  - name: windows_resource_alerts
    rules:
      - alert: HighCpuUsageWindows
        expr: |
          100 - (avg by(instance, system_name, instance_role, host) (
            rate(windows_cpu_time_total{mode="idle"}[5m])
          )) * 100 > 80
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "[Windows] CPU 사용률 80% 초과"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): CPU {{ $value | printf \"%.1f\" }}%"

      - alert: HighMemoryUsageWindows
        expr: |
          (1 - windows_memory_available_bytes / windows_cs_physical_memory_bytes) * 100 > 80
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "[Windows] 메모리 사용률 80% 초과"
          description: "[{{ $labels.system_name }}] {{ $labels.instance_role }}({{ $labels.host }}): Memory {{ $value | printf \"%.1f\" }}%"
```

### T1.5 — Alertmanager 설정 파일 작성

`**config/alertmanager/alertmanager.yml`**:

```yaml
global:
  resolve_timeout: 5m

route:
  group_by: ['system_name', 'alertname', 'severity']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  receiver: 'admin-api-webhook'

  routes:
    # Critical 알림 → 빠른 발송
    - match:
        severity: critical
      group_wait: 10s
      repeat_interval: 3m
      continue: true

    # LLM 분석 결과 알림
    - match:
        alertname: LLMLogAnalysis
      receiver: 'admin-api-webhook'

receivers:
  - name: 'admin-api-webhook'
    webhook_configs:
      - url: 'http://admin-api:8080/api/v1/alerts/receive'
        send_resolved: true
        http_config:
          timeout: 10s

# 억제 규칙: Critical 발생 시 동일 인스턴스의 Warning 억제
inhibit_rules:
  - source_match:
      severity: 'critical'
    target_match:
      severity: 'warning'
    equal: ['instance']
```
