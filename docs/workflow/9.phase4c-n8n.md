### 섹션 1: n8n 컨테이너 추가 (Server A docker-compose.yml)

Server A의 기존 docker-compose.yml에 n8n 서비스를 추가합니다.

```yaml
  n8n:
    image: n8nio/n8n:1.44.0
    container_name: aoms-n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=${N8N_USER:-admin}
      - N8N_BASIC_AUTH_PASSWORD=${N8N_PASSWORD}
      # PostgreSQL을 n8n 내부 DB로 사용 (aoms DB의 별도 스키마)
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=postgres
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_DATABASE=aoms
      - DB_POSTGRESDB_USER=aoms
      - DB_POSTGRESDB_PASSWORD=${DB_PASSWORD}
      - DB_POSTGRESDB_SCHEMA=n8n
      - N8N_HOST=0.0.0.0
      - N8N_PORT=5678
      - WEBHOOK_URL=http://${SERVER_A_IP}:5678/
      - N8N_ENCRYPTION_KEY=${N8N_ENCRYPTION_KEY}
      - GENERIC_TIMEZONE=Asia/Seoul
    volumes:
      - n8n_data:/home/node/.n8n:z
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '0.3'
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - aoms-network
```

### 섹션 4: 워크플로우 2 — 메트릭 이상 벡터 검색

Alertmanager에서 n8n Webhook으로 알림을 수신하여 유사 이력 검색 후 Teams 전달:

```
알림 흐름:
Alertmanager → [기존] Admin API :8080/api/v1/alerts/receive
                ↓
           + [추가] n8n Webhook :5678/webhook/metric-alert
                ↓
           Ollama 임베딩 → Qdrant 검색 → 유사 이력 포함 Teams
```

n8n Webhook 수신 설정:
- Path: `/webhook/metric-alert`
- Method: POST

Alertmanager webhook 설정에 n8n URL 추가:
```yaml
# alertmanager.yml receivers에 추가
receivers:
  - name: 'aoms-receiver'
    webhook_configs:
      - url: 'http://admin-api:8080/api/v1/alerts/receive'
      - url: 'http://n8n:5678/webhook/metric-alert'  # 벡터 검색용 추가
```

워크플로우 노드 순서:
1. Webhook 수신 → alerts[] 배열 처리
2. Ollama `/api/embeddings` 호출 (alert description 임베딩)
3. Qdrant `/collections/log_incidents/points/search` 검색
4. IF: 유사 이력 존재 시 → Admin API에 context 포함 추가 알림

### 섹션 5: 워크플로우 3 — 피드백 등록 처리 (Flask :8081 대체)

Flask Feedback Server를 n8n Webhook으로 완전 대체합니다.

Admin API에 피드백 폼 서빙 추가:
```python
# Admin API routes/feedback.py
@router.get("/feedback-form")
async def feedback_form(alert_id: int, system: str, request: Request):
    """Teams 알림의 '해결책 등록' 버튼 클릭 시 접근하는 HTML 폼"""
    html_content = f"""
    <html><body>
    <h2>{system} 장애 해결책 등록</h2>
    <form action="http://n8n:5678/webhook/feedback" method="POST">
      <input type="hidden" name="alert_id" value="{alert_id}">
      <input type="hidden" name="system" value="{system}">
      <select name="error_type">
        <option>DB 연결 오류</option>
        <option>메모리 부족</option>
        <option>디스크 부족</option>
        <option>네트워크 오류</option>
        <option>타임아웃</option>
        <option>기타</option>
      </select>
      <textarea name="solution" placeholder="해결 내용 상세 기술..."></textarea>
      <input name="resolver" placeholder="처리자 이름">
      <button type="submit">등록</button>
    </form>
    </body></html>
    """
    return HTMLResponse(content=html_content)
```

n8n 워크플로우 3 (피드백 처리):
```
Webhook (POST /webhook/feedback)
  → PostgreSQL: INSERT INTO alert_feedback (system_id, error_type, solution, resolver)
  → Ollama: POST /api/embeddings (solution 텍스트 임베딩)
  → Qdrant: POST /collections/log_incidents/points/payload (point_id에 resolution 추가)
  → Admin API: POST /api/v1/notifications/system ("피드백 등록 완료" Teams 알림)
```

기존 JSONL 피드백 마이그레이션:
```python
# 마이그레이션 스크립트: /opt/monitoring/scripts/migrate_feedback.py
import json, psycopg2
from pathlib import Path

FEEDBACK_FILE = "/opt/monitoring/data/alert_feedback.jsonl"
DATABASE_URL = os.environ.get("DATABASE_URL")

def migrate():
    conn = psycopg2.connect(DATABASE_URL)
    cur = conn.cursor()

    with open(FEEDBACK_FILE, encoding='utf-8') as f:
        for line in f:
            item = json.loads(line.strip())
            cur.execute("""
                SELECT id FROM systems WHERE system_name = %s
            """, (item.get('system'),))
            row = cur.fetchone()
            system_id = row[0] if row else None

            cur.execute("""
                INSERT INTO alert_feedback (system_id, error_type, solution, resolver, saved_at)
                VALUES (%s, %s, %s, %s, %s)
                ON CONFLICT DO NOTHING
            """, (system_id, item.get('error_type','기타'),
                  item.get('solution',''), item.get('resolver',''),
                  item.get('saved_at', 'NOW()')))
    conn.commit()
    print(f"마이그레이션 완료")
    conn.close()
```
