# Phase 4d — Agentic LLM 심층 분석

> **문서 버전**: v1.0
> **작성일**: 2026-03-15
> **전제조건**: Phase 4b (벡터 DB 유사도 분석) 완료
> **목표**: 즉시 알림(vector)과 자율 심층 분석(react)을 2-tier로 분리하여 긴급 대응과 정확도를 동시에 확보

---

## 개요

### 설계 배경

| 문제 | 해결 방향 |
|---|---|
| 단일 LLM 호출은 제한된 컨텍스트만 활용 | LLM이 도구를 스스로 호출하여 근거 수집 |
| ReAct 루프의 다중 호출은 긴급 장애 대응에 부적합 | vector(빠름) + react(정확) 역할 분리 |
| react 실행 중 n8n 재trigger로 중복 분석 발생 가능 | PostgreSQL TTL 잠금으로 per-system 중복 방지 |
| 도구 추가 시 재배포 필요 | PostgreSQL 기반 동적 도구 레지스트리 |

### 2-Tier 분석 구조

| Tier | 모드 | 조건 | 역할 | LLM 호출 | 소요 시간 |
|---|---|---|---|---|---|
| 1 | **vector** | 항상 실행 | 즉시 이상징후 감지 + Teams 알림 | 1회 | ~10초 |
| 2 | **react** | `AGENT_MODE_ENABLED=true` | 심층 원인 분석 + 해결책 상세 제시 (follow-up 알림) | 최대 5회 | ~60-90초 |

### main.py 핵심 변경

```python
@app.post("/analyze/trigger")
@app.post("/analyze/trigger/{system_name}")
async def analyze_trigger(
    background_tasks: BackgroundTasks,
    system_name: Optional[str] = None
):
    systems = [system_name] if system_name else await get_all_active_systems()

    for sys in systems:
        # Tier 1: vector 항상 즉시 실행
        result = await vector_analyzer.run(sys)
        await notify_and_store(result, mode="vector")

        # Tier 2: react 옵션 (백그라운드)
        if settings.AGENT_MODE_ENABLED:
            background_tasks.add_task(run_react_analysis, sys)

    return {
        "status": "ok",
        "mode": "vector" + ("+react" if settings.AGENT_MODE_ENABLED else "")
    }


async def run_react_analysis(system_name: str):
    """BackgroundTask: react 심층 분석"""
    acquired = await react_lock.try_acquire(system_name, ttl_seconds=settings.AGENT_LOCK_TTL_SECONDS)
    if not acquired:
        logger.info(f"[{system_name}] react 실행 중, 건너뜀")
        return
    try:
        result = await agent_analyzer.run(system_name)
        await notify_and_store(result, mode="react")
    except Exception as e:
        logger.error(f"[{system_name}] react 오류: {e}")
    finally:
        await react_lock.release(system_name)
```

## T4d.3 — 동적 도구 레지스트리

### 도구 로딩 (agent_tools.py)

```python
from dataclasses import dataclass
from typing import Any

@dataclass
class AgentToolSpec:
    name: str
    description: str
    parameters_desc: str
    instance: Any   # PrometheusQueryTool | LokiQueryTool | VectorSearchTool

# Python 클래스 레지스트리 (새 도구 추가 시 여기에만 등록)
TOOL_CLASS_MAP: dict[str, type] = {
    "PrometheusQueryTool": PrometheusQueryTool,
    "LokiQueryTool":       LokiQueryTool,
    "VectorSearchTool":    VectorSearchTool,
}

async def load_tools_from_db(db_pool) -> list[AgentToolSpec]:
    """
    agent_tools 테이블에서 enabled=true 도구 로드.
    분석 시작마다 호출 → DB 변경 즉시 반영, 재배포 불필요.
    """
    rows = await db_pool.fetch(
        "SELECT name, description, parameters_desc, tool_class, config "
        "FROM agent_tools WHERE enabled = TRUE ORDER BY id"
    )
    tools = []
    for row in rows:
        cls = TOOL_CLASS_MAP.get(row["tool_class"])
        if cls is None:
            logger.warning(f"Unknown tool_class '{row['tool_class']}', skipping")
            continue
        tools.append(AgentToolSpec(
            name=row["name"],
            description=row["description"],
            parameters_desc=row["parameters_desc"],
            instance=cls(config=row["config"])
        ))
    return tools
```

## T4d.4 — 프롬프트 기반 ReAct 루프

### 설계 원칙

내부 LLM API가 OpenAI `tools` 파라미터를 지원하지 않을 수 있으므로,
**프롬프트 기반 ReAct** 방식을 기본으로 사용. 텍스트 생성 API면 모두 동작.

> 옵션: `AGENT_NATIVE_TOOLS=true` 설정 시 OpenAI `tools` 파라미터 방식으로 전환 가능.

### LLM 출력 포맷

```
Thought: CPU 사용률이 급증했는지 확인해야 한다
Action: query_prometheus
Action Input: {"query": "rate(node_cpu_seconds_total{mode=\"idle\",system_name=\"pos-service\"}[5m])", "start": "10m"}
```
→ 코드가 도구 실행 후 추가:
```
Observation: {"avg_idle_ratio": 0.12, "trend": "급감", "peak_time": "09:42", "values": [...]}
```
→ 다음 반복...
```
Thought: CPU가 88%까지 치솟았다. 오류 로그를 확인하겠다
Action: query_loki_logs
Action Input: {"query": "{system_name=\"pos-service\"} |~ \"Exception|OOM|GC\"", "start_offset_minutes": 15}
```
→ ...반복...
```
Final Answer: {
  "severity": "CRITICAL",
  "root_cause": "JVM 힙 부족으로 인한 Full GC 폭증 → CPU 급증",
  "evidence": ["CPU idle 12% (09:42)", "GC overhead limit exceeded 37건", "유사 이력 2건 (2025-11, 2026-01)"],
  "recommendation": "즉시: JVM 재시작. 단기: 힙 메모리 -Xmx 증설 (현재 2GB → 4GB). 장기: 메모리 누수 패턴 코드 리뷰",
  "error_category": "MEMORY",
  "anomaly_type": "recurring",
  "confidence": "high"
}
```

### 파싱 정규식

```python
import re

THOUGHT_RE = re.compile(r"Thought:\s*(.+?)(?=Action:|Final Answer:|$)", re.DOTALL)
ACTION_RE  = re.compile(r"Action:\s*(\w+)")
INPUT_RE   = re.compile(r"Action Input:\s*(\{.+?\})", re.DOTALL)
FINAL_RE   = re.compile(r"Final Answer:\s*(\{.+\})", re.DOTALL | re.IGNORECASE)
```

### 시스템 프롬프트 (도구 목록 동적 삽입)

```python

## 사용 가능한 도구
{tool_section}

## 출력 형식 (반드시 준수)

Action: <도구명>
Action Input: <JSON 파라미터>

## 최종 출력 형식 (충분한 증거 수집 후)
```

### ReAct 루프 구현 (agent_analyzer.py)

```python
async def _react_loop(
    system_name: str,
    trigger_context: dict,
    tools: list[AgentToolSpec]
) -> AgentResult:
    system_prompt = build_system_prompt(tools)
    initial_user_msg = build_initial_context_message(system_name, trigger_context)
    messages = [
        {"role": "system",  "content": system_prompt},
        {"role": "user",    "content": initial_user_msg}
    ]
    tool_map = {t.name: t.instance for t in tools}
    iterations = 0
    tool_calls_made = []
    start_time = time.monotonic()

    while iterations < config.max_iterations:
        iterations += 1

        # 토큰 예산 초과 시 오래된 Observation 제거
        if not token_budget.check_budget(messages, config.token_budget):
            messages = token_budget.trim_oldest_observations(messages, keep_last=2)

        response = await llm_client.chat_completions(
            messages=messages,
            max_tokens=1024
        )
        content = response["choices"][0]["message"]["content"]
        messages.append({"role": "assistant", "content": content})

        # ① Final Answer 감지 → 정상 종료
        final_match = FINAL_RE.search(content)
        if final_match:
            final_analysis = json.loads(final_match.group(1))
            break

        # ② Action 파싱
        action_match = ACTION_RE.search(content)
        input_match  = INPUT_RE.search(content)
        if not action_match:
            # LLM이 포맷을 무시 → best-effort 텍스트 파싱 후 종료
            final_analysis = _parse_best_effort(content)
            break

        tool_name = action_match.group(1)
        tool_args = json.loads(input_match.group(1)) if input_match else {}
        tool_calls_made.append(tool_name)

        # ③ 도구 실행
        tool_instance = tool_map.get(tool_name)
        if tool_instance is None:
            observation = json.dumps({"error": f"unknown_tool: {tool_name}"})
        else:
            try:
                raw = await asyncio.wait_for(
                    tool_instance.execute(**tool_args),
                    timeout=config.tool_timeout_seconds
                )
                observation = token_budget.truncate(json.dumps(raw), max_chars=config.tool_result_max_chars)
            except asyncio.TimeoutError:
                observation = json.dumps({"error": "tool_timeout", "tool": tool_name})
            except Exception as e:
                observation = json.dumps({"error": str(e), "tool": tool_name})

        messages.append({"role": "user", "content": f"Observation: {observation}"})

    else:
        # max_iterations 도달: 마지막 응답에서 best-effort 파싱
        final_analysis = _parse_best_effort(messages[-2]["content"])  # 마지막 assistant 메시지

    return AgentResult(
        system_name=system_name,
        mode="react",
        iterations_used=iterations,
        tool_calls_made=tool_calls_made,
        final_analysis=final_analysis,
        token_usage=response.get("usage", {}),
        duration_seconds=time.monotonic() - start_time
    )
```

### 루프 종료 조건 (우선순위 순)

| 우선순위 | 조건 | 처리 |
|---|---|---|
| 1 | `Final Answer:` 정규식 매칭 | 정상 종료, JSON 파싱 |
| 2 | `Action:` 파싱 실패 | best-effort 텍스트 추출 후 종료 |
| 3 | `iterations == max_iterations` | 마지막 응답 best-effort 파싱 후 종료 |
| 4 | 토큰 예산 trim 후에도 초과 | 가용 정보로 즉시 종료 |
| 5 | 예외 발생 | finally에서 lock 해제 + 오류 로그 |

### react_analysis_locks 테이블 활용

```python
# react_lock.py

async def try_acquire(system_name: str, ttl_seconds: int = 300) -> bool:
    """
    잠금 획득 시도. 원자적 INSERT ... ON CONFLICT 사용.
    유효한 잠금이 이미 있으면 False 반환.
    """
    result = await db.execute("""
        INSERT INTO react_analysis_locks (system_name, locked_at, expires_at)
        VALUES ($1, NOW(), NOW() + ($2 * INTERVAL '1 second'))
        ON CONFLICT (system_name) DO UPDATE
          SET locked_at = EXCLUDED.locked_at,
              expires_at = EXCLUDED.expires_at
          WHERE react_analysis_locks.expires_at < NOW()
        RETURNING system_name
    """, system_name, ttl_seconds)

    # RETURNING이 없으면 → 유효한 잠금 존재 (ON CONFLICT 갱신 안 됨)
    return result is not None

async def release(system_name: str):
    await db.execute(
        "DELETE FROM react_analysis_locks WHERE system_name = $1",
        system_name
    )
```

### 시나리오별 동작

```
[시나리오 1] 정상 (4분 30초 안에 완료)
  T+0:00  n8n trigger → Tier1 vector ✓ → Tier2 react 시작 (lock 획득, TTL=300s)
  T+2:30  react 완료 → lock 해제
  T+5:00  n8n trigger → Tier1 vector ✓ → react 새로 시작 (lock 없음)  ✓

[시나리오 2] 지연 (5분 이상 소요)
  T+0:00  n8n trigger → Tier1 vector ✓ → Tier2 react 시작 (lock 획득)
  T+5:00  n8n trigger → Tier1 vector ✓ → react lock 존재 → SKIP  ✓
           (담당자는 즉시 알림 수신, react follow-up은 완료 시 수신)
  T+6:30  react 완료 → lock 해제
  T+10:00 n8n trigger → 정상 실행  ✓

[시나리오 3] 예외 발생 (react 비정상 종료)
  T+0:00  react 시작 (lock 획득)
  T+2:00  react 예외 → finally 블록에서 lock 강제 해제  ✓
  T+5:00  n8n trigger → react 정상 시작  ✓

[시나리오 4] 서버 재시작 (in-memory lock과 달리 복구 불필요)
  재시작 후 → expires_at < NOW() 인 잠금은 ON CONFLICT 조건으로 자동 덮어씀  ✓
```

### TTL 계산 기준

```
AGENT_LOCK_TTL_SECONDS 기본값 = AGENT_MAX_ITERATIONS × AGENT_TOOL_TIMEOUT_SECONDS × 2
기본 계산: 5회 × 15초 × 2 = 150초
권장값:    300초 (여유 있는 안전 마진)
```
